                     TEST DRIVEN DEVELOPMENT (TDD)

WHAT IS TDD?
===============================================================================

TDD stands for Test Driven Development.

It is a software development methodology in which
tests are written BEFORE writing the actual code.

Instead of writing code first,
developers write a failing test,
then write the minimum code to pass the test,
and finally improve the code.

===============================================================================
TDD CYCLE
===============================================================================

RED

↓

GREEN

↓

REFACTOR

Repeat

===============================================================================
RED PHASE
===============================================================================

Write a Test.

There is no implementation.

Result

FAILED

===============================================================================
GREEN PHASE
===============================================================================

Write minimum code.

Result

PASSED

===============================================================================
REFACTOR PHASE
===============================================================================

Improve the code.

Do not change functionality.

Result

PASSED

===============================================================================
ADVANTAGES
===============================================================================

• Better Code Quality

• Fewer Bugs

• High Test Coverage

• Easier Refactoring

• Cleaner Design

• Faster Debugging
